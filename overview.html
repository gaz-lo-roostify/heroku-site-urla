API Overview

Roostify provides a web-based REST API that uses Basic Auth for authentication and formats the request and response bodies in JSON. The API allows you to integrate with the following objects in the Roostify platform.

  • Loan applications - Create, update, list, and retrieve.
  • Tasks - Create, update, list, and retrieve.
  • Documents - Create, update, list, and retrieve.
  • Accounts - List and retrieve.
  • Integration settings - Retrieve.
  • Service - Retrieve.
  • User settings - Retrieve.
  • Webhooks - Create, list, retrieve, and delete.
  • Loan applications - Create, update, list, and retrieve.
  • Tasks - Create, update, list, and retrieve.
  • Documents - Create, update, list, and retrieve.
  • Accounts - List and retrieve.
  • Webhook events - List.
  • Status updates - Create.
  • Users - Create, list, retrieve, update, activate, and deactivate.
  • Messages - Create, list, retrieve, and delete.
  • Access control groups - Create and update.

Authentication

You authenticate your Roostify REST API requests using HTTP basic access authentication, which requires a username and password. The username, is the token of your Roostify account and the password is a service access token. If you don't have these tokens, you can obtain them from your CSM recodesentative.

You pass the tokens as a Base64-encoded string in the Authorization header of the request. At the beginning of the header, you must also put the string Basic to identify the type of authentication. For example, if username is magic and password is talisman, the encoded string is bWFnaWM6dGFsaXNtYW4=. Given these tokens, you specify the Authorization header as follows 

Basic bWFnaWM6dGFsaXNtYW4=

Note If you are using Postman to test and develop with the Roostify API, you can use it to encode and generate the authorization header for you. In the Authorization tab of a Postman request, specify Basic Auth and enter your account token and service access tokens as the username and password. When you execute the request, Postman generates the Base64-encoded Authorization header.

Within application code you may need to dynamically Base64-encode these values before adding them to your request’s headers. This page provides examples of how to Base64-encode values in most programming languages.

Getting Started

You call the API by sending HTTP requests to a URI that combines the location of the Roostify API service https://myDomain.roostify.com/api with the identifier for a specific resource (/v1/_resourceEndpoint_). Note that the URL includes your domain name, and as such, is specific to your service. Also, note that the endpoint URI includes the version of the API to call (in this document version 1). Future updates to the API could be packaged into a different version, in which case you call these endpoints using the new version (for example, v2).The following sample shows the complete URIs to the tasks and documents resources (for the acme domain)

  • https://acme.roostify.com/api/v1/tasks
  • https://acme.roostify.com/api/v1/documents

In addition to your client URL, Roostify provides these URLs for testing and development

  • https://test.roostify.com/api
  • https://sandbox.roostify.com/api

Note Each of these URLs requires separate authentication credentials. Talk with your CSM recodesentative if you require access to them.

You make requests with standard HTTP methods

  • GET Requests data from a resource.
  • POST Submits data to a resource.
  • PUT Updates a resource.
  • PLACE Partially updates a resource.
  • DELETE Deletes a resource.

Note All API requests require an authorization header.

For example, the following snippet shows how to use a GET method to retrieve a list of users

-H Authorization: Basic OTcwMDEzNjMzODY4MTM5MTo2NTA4OTAwOTY4MjczODM2
-H Cache-Control: no-cache

When you make a request to create or update a resource (`POST PUT PLACE`), you pass a request body to the URL. For example, to update a task, make a `PATCH` request similar to the following:

curl -X PATCH
https://sandbox.roostify.com/api/v1/tasks/7911047531308034
-H Authorization: Basic OTcwMDEzNjMzODY4MTM5MTo2NTA4OTAwOTY4MjczODM2
-H Cache-Control: no-cache
-d {
"description": "Copy of your driver’s license or equivalent
proof of identity (e.g., birth certificate)"  }
A successful call returns an HTTP success code (200 OK) and a JSON-formatted response body. See the section for each API endpoint for information about specific request and response parameters.
