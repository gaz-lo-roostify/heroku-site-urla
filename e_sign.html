Creating an eSign Document Using the API



Integrating eSign documents into the loan application process helps you to streamline the mortgage process. An eSign document prompts the user to enter signatures by clicking a button, and tracks when all required signatures are entered.

You can add an eSign document to an existing loan application or create a loan application and then create an eSign document to insert into it.

Follow these steps to add an eSign document to a loan application using Postman.


Use a GET request as shown in the image or similar to retrieve the details for a loan application. (Hover over images to zoom)


Copy and note down the following from the loan application that you retrieved:

  • loan application id
  • borrower reference id
  • co borrower reference id


For example:





Create your POST request to add the eSign document by using the following template, shown here with sample values:




{
    "loan_application_id": "5031663310293454",
    "name": "DocuSign Package 5ß",
    "assigned_role": "borrower",
    "extension_type": "docusign",
    "description": "e-sign task-2",
    "documents": [
      {
      "template_name": "e-sign task-1",
      "required_signers": ["jeff12","jeff221"],
      "file_name": "Request For Signature",
      "file_content": “JVBERi0xLjQKJeLjz9MKMSAwIG9iaiA8PC9M….”
}
    ]



In your POST to create the eSign document:

  • Replace the loan_application_id value in the template shown above with your loan application id
  • Replace the required_signers values with the b_reference_id and c_reference_id from your loan application as described above
